# GW-FOC-Lab 任务日志 / Task Log

> 项目：GW-FOC-Lab  
> 用途：记录每周/每日实际推进情况、TODO、Deadline、输出物和复盘  
> 原则：任务要小，必须有输出物；没有输出物的讨论不算完成；忙碌周也至少留下一条记录。  
> 建议文件位置：`docs/task_log.md`

---

## 1. 使用说明

这个文件用于记录我们两个人在 GW-FOC-Lab 项目中实际做过的事情。它不是用来制造压力的，而是用来防止项目“聊过但没做、做过但没记录、记录了但不可展示”。

每条任务尽量满足以下要求：

1. 任务足够小，最好 0.5～3 小时内可以完成。
2. 每条任务必须有明确输出物，例如代码、截图、实验数据、文档、视频、曲线图或问题记录。
3. Deadline 不是为了催命，而是为了提醒任务不要无限拖延。
4. 如果任务没有完成，不要删除，直接改状态并写清楚原因。
5. 每周至少更新一次本文件。

---

## 2. 状态说明

| 状态 | 含义 |
|---|---|
| TODO | 已确定要做，但还没开始 |
| DOING | 正在做 |
| BLOCKED | 被问题卡住，需要讨论或等待条件 |
| DONE | 已完成，并且有输出物 |
| PAUSED | 暂时搁置，后续可能继续 |
| CANCELED | 确认不再做 |
